导入数据

General 格式

使用 dx 语言风格的头文件, 可以指定需要读取的文件, 其内容不限于矩阵数据，还包括各种列表和其他风格的数据, 其中头文件以 .general 为后缀.
比如一个简单的数据文件:

我们可以打开GUI,选择导入数据，并指定 General 格式,通过GUI可以设定网格，比如选择 regular 的网格, 指定数据是 a.data, 并 skip 数据的第一行, 网格尺寸 5*5 , 保存这个 general 头文件, 选择显示数据, dx 会通过默认的网络画出图片.

重新打开 .general 文件, 大致可以发现该文件使用 keyword statement 的方法描述数据.

每行都有一个关键词, 比如 file 关键词 = a.data, 其中 positions 关键词比较特殊, 它描述了网格顶点的信息, 特殊之处在于它可以是多行的.末尾用 end 结束.注释是井号.

dx 格式

我们还可以直接用 .dx 格式的头文件来描述数据, 这也是最常使用的方式.比如对上面那个 a.data 的例子，用 .dx 文件也可以比较方便地描述.

首先我们需要知道，dx 中的数据由一些 field 组成, 每个 field 含有许多个组件(component), 这些组件可以是数组(Array), 也可以是本身定义好的一些网格结构, 这些组件或者场都叫做对象(Object),有自己定义的别名.

有用的模块

图形界面特有模块

每个模块可以拖拽，有许多 tab 附着其上, 有些模块可以增加 tab , 其实际含义是输入与输出，双击可查看实际含义.

Image 模块用于自动渲染显示图片, Transmitter/Reciver 专门用于管理图形编程时不同 page 间的相互联系.

除此之外,其余的模块基本都能在脚本(script)中使用,打开命令行 dx 脚本:

基本的画图模块

Import
导入指定格式的数据, 手动输入数据文件名或者通过 FileSelector 模块导入名字.

AutoColor
为数据自动设色,相当于

RubberSheet
曲面, 用于为二维网格上的数据画三维图.

Isosurface
等值面, 可设定等值或等值面个数

其中 value 可以用 1,3,5 或 1 .. 5:2 这种写法.

ShowConnections
展现网格结构.

ColorBar
色条

AutoCamera
相机(方向角)，自动.

Render
渲染

Display
展示

WriteImage
保存图片

Collect
合并一些对象

Format

模板比如”sol-%d.dx”, 类似于 C++ 中的语法, %d 表示整数值, 该值由 value 输入,它可能是一个 list 或者 sequencer.

Compute
逐点计算(对 data 组分操作,可通过 Mark,Unmark 使其对其他组分操作)

Caption
标题,可指定标题位置

ShowBox

Gradient
梯度

Map
将一些值映射到一些值

Tube
加粗线条

Post
改变数据的依赖(位置或者网格)

Sequencer
动画序列
